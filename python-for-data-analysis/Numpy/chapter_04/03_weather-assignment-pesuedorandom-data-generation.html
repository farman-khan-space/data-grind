Random Data Generation & Universal Functions

Case Study: New York City Climate Model

You're working with a team of climate scientists studying weather patterns in New York City. They need to test their analysis software, but they are waiting for the complete, cleaned dataset from the meteorological department.

To avoid delays, your task is to **simulate a year's worth of weather data**. You'll generate realistic-looking temperature data, clean it up, transform it, and perform some initial analysis. This project will heavily rely on your skills with NumPy's random number generator and its powerful Universal Functions (ufuncs).

Learning: Core Intuitions

What is Pseudorandom? (And Why "Seed" is Important)

You're right, the book is brief here. Let's use an analogy: a deck of cards.

  • ➜
    No Seed (True Randomness): Every time you run your code, it's like getting a brand new, perfectly shuffled deck of cards. The sequence of cards you draw is unpredictable. This is what happens if you don't use `np.random.seed()`.
  • ➜
    Using a Seed (Pseudorandom): The seed is like a secret number that arranges the deck in a specific, known order *before* shuffling. For example, `seed(123)` might mean "arrange all the hearts first, then spades...". The shuffling algorithm itself is deterministic (not random). So, if you and I both use the same seed (the same arrangement) and the same shuffle, we will get the **exact same sequence of "random" cards**.

This is why we use it: for reproducibility. It guarantees that your "random" data is the same every time you run the script, which is essential for testing and sharing your work.

What are Universal Functions (ufuncs)?

Think of ufuncs as super-charged math operations designed specifically for NumPy arrays.

  • ➜
    The Slow Way (Python `for` loop): This is like using a basic calculator. To add 1 to a million numbers, you have to punch in `number + 1`, press equals, write it down, and repeat a million times. It's slow because Python is checking types and handling things one by one.
  • ➜
    The Fast Way (NumPy ufunc): This is like a spreadsheet formula. You tell NumPy, "take this entire array of a million numbers and add 1 to all of them." NumPy then uses its highly optimized, low-level code (written in C) to perform the operation on the entire block of memory at once. It's thousands of times faster. `np.sqrt()`, `np.exp()`, and even basic math like `+` and `*` on arrays are all ufuncs.

Your Simulation Challenge

1. Generate the Temperature Data

Let's simulate one year (365 days) of average daily temperatures. We'll assume the average temperature follows a normal (Gaussian) distribution. For NYC, let's use a mean temperature of 13°C and a standard deviation of 8°C.

# 1. First, set the random seed to 42 for reproducibility.
# 2. Use `np.random.normal()` to generate 365 temperatures.
# - `loc` (mean) = 13
# - `scale` (std dev) = 8
# - `size` = 365
# 3. Store this in an array called `temperatures_celsius`.
# 4. Print the first 5 values.

2. Clean the Data with `np.where`

Due to a simulated sensor glitch and natural weather patterns, some impossible/extreme temperatures (below 0°C) might have been generated. Your task is to clean these by setting a floor value. `np.where` is a vectorized version of an `if/else` statement and is perfect for this.

# 1. Use `np.where(condition, value_if_true, value_if_false)`.
# 2. The condition should be `temperatures_celsius < 0`.
# 3. If the condition is true, replace the value with 0 (a floor value).
# 4. If false, keep the original value.
# 5. Store the result back in `temperatures_celsius` and print the `.min()` to verify.

3. Convert to Fahrenheit (Unary ufuncs)

The international team prefers Fahrenheit. Convert the entire Celsius array using the formula: `F = C * 9/5 + 32`. This is a classic use of element-wise arithmetic ufuncs.

# 1. Apply the conversion formula directly to the `temperatures_celsius` array.
# 2. Store the result in a new array called `temperatures_fahrenheit`.
# 3. Print the first 5 values of the new Fahrenheit array.

4. Calculate Heat Index (Binary ufuncs)

Heat index combines temperature and humidity. Let's simulate humidity and calculate a simplified heat index. This requires operations between two separate arrays.

# 1. Generate 365 humidity values using `np.random.uniform(low=0.4, high=0.9, size=365)`. Store in `humidity`.
# 2. Calculate a simplified heat index: `heat_index = temperatures_fahrenheit + (humidity * 10)`.
# 3. Some days might be extra hot. Use `np.maximum()` to ensure no heat index value is below the actual Fahrenheit temperature. Compare `heat_index` and `temperatures_fahrenheit` and store the result back in `heat_index`.
# 4. Print the first 5 heat index values.

Solutions

1. Generate Temperature Data

import numpy as np
# 1. Set the seed
np.random.seed(42)

# 2. Generate the data
temperatures_celsius = np.random.normal(loc=13, scale=8, size=365)

# 3. Print the first 5 values
print(temperatures_celsius[:5])
# Expected output: [16.9737  8.5816 18.7854 25.3588 11.9069]

2. Clean the Data with `np.where`

# The np.where function acts on the whole array at once.
temperatures_celsius = np.where(temperatures_celsius < 0, 0, temperatures_celsius)

# Verify by checking the minimum value
print(f"Minimum temperature after cleaning: {temperatures_celsius.min():.4f}°C")
# Expected output: Minimum temperature after cleaning: 0.0000°C

3. Convert to Fahrenheit

# The formula is applied element-wise to the entire array.
temperatures_fahrenheit = temperatures_celsius * (9/5) + 32

print(temperatures_fahrenheit[:5])
# Expected output: [62.5526 47.4469 65.8138 77.6458 53.4325]

4. Calculate Heat Index

# Ensure you use the same seed to get the same humidity values
np.random.seed(42)
humidity = np.random.uniform(low=0.4, high=0.9, size=365)

# Calculate the initial heat index
heat_index = temperatures_fahrenheit + (humidity * 10)

# Use the binary ufunc np.maximum to compare two arrays element-wise
heat_index = np.maximum(heat_index, temperatures_fahrenheit)

print(heat_index[:5])
# Expected output: [69.5063 52.2198 72.3094 85.3894 60.8819]
